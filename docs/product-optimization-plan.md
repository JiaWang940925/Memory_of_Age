# 岁语产品优化整理

## 0. 已确认的产品前提

以下方向已经确认，后续方案统一按这些前提展开：

- 第一阶段核心付费人群：40-60 岁子女
- 第一阶段核心陪伴对象：老人本人
- 产品形态：普通家用产品
- 当前不公开使用“阿尔茨海默预防”类表述
- 主要设备：手机
- 允许“老人 + 子女”共同使用同一个账号

由此推导出的产品策略是：

- 商业上优先打动子女
- 体验上优先照顾老人
- 交互上必须支持家庭协同
- 设计上必须是移动端优先，而不是桌面优先
- 文案上以“回忆录 + 脑健康陪伴”替代疾病导向表达

## 1. 当前产品现状

当前版本已经具备以下能力：

- 通过资料建档生成个性化访谈问题
- 通过对话记录人生故事与照片
- 将故事整理为回忆录与可打印 PDF
- 根据年龄、地域、故事内容生成时代线索
- 在地图页标记重要人物、地点与回忆节点
- 提供“日常回想”入口，作为轻量回忆陪伴功能

当前版本的主要问题不是功能不够，而是产品主线还不够清晰：

- 面向老人本人和面向子女代访，混在同一条路径里
- “写回忆录”和“认知陪伴”混在一起，用户难以理解
- “阿尔茨海默预防”这一定位过重，但当前能力更接近回忆陪伴与认知活化
- 日常回想逻辑偏“答题校验”，对老年人来说压力偏大

## 2. 建议的产品定位

建议把产品定位明确为：

**一款面向老年人与家属协同使用的“回忆录整理 + 脑健康陪伴”产品。**

更稳妥的对外表达建议：

- 主定位：帮助老人留下人生故事，帮助家人了解父母过往经历
- 副定位：通过温和、持续的回想活动，支持老年人的认知活化与日常陪伴
- 不建议直接承诺：预防阿尔茨海默、筛查阿尔茨海默、判断是否患病

一句话版本可整理为：

**岁语是一款帮助老人讲述人生、帮助家人整理家庭记忆，并通过日常回想支持脑健康陪伴的产品。**

## 3. 核心用户分层

建议明确两类主用户，而不是只定义成“老人产品”。

同时需要区分：

- 谁是主要使用对象
- 谁是主要付费对象
- 谁是主要操作者

### 3.1 老年用户

需求重点：

- 想有人耐心听自己讲故事
- 想把过去的人和事留下来
- 容易忘词、容易疲惫，不适合复杂操作
- 更需要鼓励、提示，而不是对错判断

适合的产品价值：

- 温和访谈
- 语音优先
- 老照片触发记忆
- 回忆地图与人物关系帮助继续回想
- 日常轻量回想陪伴

这是第一阶段最核心的“被服务者”。

### 3.2 中年子女用户

需求重点：

- 想了解父母年轻时经历
- 想替父母整理回忆录
- 想保存家庭故事、人物关系和照片
- 想观察父母最近是否出现明显记忆退化

适合的产品价值：

- 家属代访模式
- 家庭档案整理
- 重点人物与关系校正
- 代为补充地点、照片、事件
- 日常陪伴记录与趋势观察

这是第一阶段最核心的“付费者”和“推动使用的人”。

### 3.3 第一阶段的产品判断

第一阶段不建议把产品做成“完全由老人独立完成”的工具，也不建议做成“完全由子女代办”的工具。

更合理的方向是：

- 由子女发起使用
- 由老人获得陪伴体验
- 由双方共同沉淀家庭记忆

一句话理解：

**买单的人是子女，真正感受到温度的人是老人。**

## 4. 建议的产品结构

建议把现有产品拆成两条清晰主线。

同时整体产品形态建议明确为：

- `家用产品`
- `移动端优先`
- `家庭共用账号`

### 4.1 主线 A：人生回忆录

目标：

- 让老人讲故事
- 让家属整理故事
- 输出可保存、可打印、可传承的家庭记忆资产

建议包含：

- 资料建档
- 智能访谈
- 回忆录整理
- 地图与人物关系图
- PDF 导出

### 4.2 主线 B：日常回想陪伴

目标：

- 作为低压力、可持续的日常认知活化工具
- 帮助家属建立陪伴节奏
- 为老人提供熟悉人物、地点、生活线索的回想入口

建议包含：

- 每日轻量回想
- 生活触发器卡片
- 家属陪练模式
- 简单趋势记录
- 温和异常提醒

## 5. 首页与路径优化建议

当前首页入口建议改成三选一，而不是只给“开始我的故事”和“日常回想”。

建议首页三个入口：

1. 我来记录自己的人生
2. 我来帮父母整理回忆
3. 今天做一次轻松回想

这样做的好处：

- 老人本人不会被“代访”逻辑干扰
- 子女会知道这是允许协同使用的产品
- 日常回想从附属功能变成独立场景

首页文案策略建议：

- 对子女说：帮父母留下故事，趁还能问的时候多问一点
- 对老人说：有人愿意认真听您讲过去的故事
- 对家庭说：一起把家里的记忆整理下来

## 6. 资料页优化建议

当前资料页主要记录老人基础信息。建议新增一层“使用关系信息”。

建议新增字段：

- 当前操作者是谁：本人 / 子女 / 其他家属 / 护理人员
- 与记录对象关系：儿子 / 女儿 / 儿媳 / 女婿 / 孙辈 / 护理员
- 是否本人在场
- 是否允许家属补充记录
- 家庭常用称呼

建议保留并强化的字段：

- 出生年月日
- 出生地
- 成长地
- 性别
- 常用称呼

建议新增的“记忆触发信息”字段：

- 曾长期工作或生活的地方
- 家庭成员称呼
- 重要职业或身份
- 童年常住环境
- 常提到的老物件或兴趣

目的：

- 让后续问题更贴近个人
- 提前为人物、地点、地图、日常回想提供素材

基于“共同账号”前提，资料页还应新增：

- 谁创建了这个家庭档案
- 是否允许家属继续编辑
- 是否允许家属替老人补充照片和备注

## 7. 访谈功能优化建议

当前访谈逻辑已经有“人生阶段问题 + 时代事件提示”，方向是对的，但还需要按场景分模式。

建议拆成三种访谈模式：

### 7.1 自述模式

适合老人本人直接使用。

特点：

- 一次一个问题
- 支持长按语音
- 提示语言更慢、更温和
- 少输入、多口述

### 7.2 家属陪访模式

适合子女陪父母一起回忆。

特点：

- 问题同时显示“老人版提问”和“家属版追问建议”
- 允许家属补充备注
- 允许家属标记“这个人是谁”“这张照片是谁”
- 允许把回答拆成多人共同补充
- 允许子女先准备问题，再在陪聊时切给老人回答

### 7.3 轻量认知回想模式

适合日常重复使用。

特点：

- 不强调完整故事
- 重点回想熟悉人物、地方、日常细节
- 更像聊天，不像正式访谈

## 8. 记忆触发器建议

目前产品的触发器主要是时代大事件、人物和地点。建议扩成一套更适合老人回想的“生活触发器体系”。

建议新增五类触发器：

### 8.1 人物触发器

- 父母
- 伴侣
- 子女
- 孙辈
- 老师
- 同事
- 邻居

### 8.2 地点触发器

- 老家
- 学校
- 工作单位
- 集市
- 火车站
- 河边
- 医院

### 8.3 物件触发器

- 缝纫机
- 自行车
- 收音机
- 黑白电视
- 搪瓷缸
- 饭盒
- 工作证

### 8.4 生活场景触发器

- 过年
- 做饭
- 上学路上
- 上班早班车
- 家庭聚餐
- 婚礼
- 照顾孩子

### 8.5 时代与社会触发器

- 恢复高考
- 改革开放
- 粮票布票
- 香港回归
- 神舟飞天

建议的优先级：

- P0：人物、地点、时代
- P1：物件、生活场景

对家用场景来说，建议优先补“家庭常见触发器”而不是宏大社会事件：

- 老照片
- 家庭称呼
- 吃饭场景
- 家里摆设
- 年节习惯
- 常见旧物

## 9. 日常回想功能优化建议

这是最需要重新定义的一块。

当前问题：

- 当前逻辑存在 `expectedAnswer` 和相似度判断
- 容易让老人觉得自己在“做题”
- 容易形成答错压力

建议改成“陪伴式回想”，不强调答案对错。

基于“手机优先”的前提，日常回想建议设计成单手可操作：

- 一个题目
- 一个提示按钮
- 一个录音按钮
- 一个“今天先到这里”按钮

### 9.1 建议的题型

- 今天想起一个家人：这是谁？
- 今天想起一个地方：这是哪里？
- 今天想起一件旧物：以前怎么用它？
- 今天想起一段日常：那时候谁常和您在一起？
- 今天想起一张照片：照片里的人和故事是什么？

### 9.2 建议的记录方式

用以下状态代替“答对率”：

- 自己很快说出
- 提示后说出
- 家属补充后想起
- 今天不想说也可以

### 9.3 建议的趋势观察

只做非常温和的趋势记录：

- 最近更容易回想的人物类型
- 最近更容易卡住的内容类型
- 连续几天对核心家庭信息明显混淆

不建议输出：

- 风险评分
- 疑似患病判断
- 诊断性结论

## 10. 地图与人物关系功能优化建议

当前地图页已经很有产品特色，建议把它从“展示页”升级成“编辑协作页”。

建议新增能力：

- 家属手动校正人物关系
- 给人物补照片
- 给人物补别名和称呼
- 给地点补具体说明
- 给节点补“这是谁讲的”
- 标记“这段回忆家属已确认”

建议新增展示层：

- 家庭关系层
- 人生阶段层
- 地点迁移层
- 时代线索层

这样地图会从“好看”升级为“能沉淀家庭记忆资产”。

考虑到第一阶段是家用产品，地图页应更偏“家庭协同整理工具”，而不是公共展示工具。

## 11. 回忆录与导出优化建议

当前回忆录导出已经升级成连贯文章，这是正确方向。下一步建议增强“给家属阅读”的能力。

建议新增导出内容：

- 人物索引页
- 地点索引页
- 时代线索索引页
- 照片附录页
- 家属补充备注页

建议新增导出版本：

- 老人阅读版：字号更大、内容更短
- 家庭保存版：正文更完整
- 家属采访版：留出追问空白区域

考虑到主要设备是手机，建议移动端优先做：

- 一键分享给家庭群的长图版摘要
- 家属收藏版短链接页面
- 手机端可连续浏览的人物 / 地点索引

## 12. 医学与风险边界建议

这一部分必须明确。

建议对外统一使用以下表述：

- 脑健康陪伴
- 日常回想支持
- 认知活化练习
- 家属陪伴式回忆

不建议直接使用：

- 预防阿尔茨海默
- 检测阿尔茨海默
- 评估是否痴呆
- 判断认知障碍等级

建议在产品中保留的边界提示：

- 本产品不是医疗器械
- 不提供医学诊断
- 若近期明显出现记忆、时间、人物识别混乱，应由家属陪同并咨询专业医生

## 13. 版本优先级建议

### P0：先把主线讲清楚，并适配家用移动端

- 首页拆成三入口
- 新增家属代访模式
- 日常回想去掉“答题感”
- 文案统一成“回忆录 + 脑健康陪伴”
- 账号改成家庭共用视角
- 关键页面按手机优先重排

### P1：增强家庭协同

- 资料页增加关系信息
- 人物手动校正
- 地点和照片补录
- 家属备注与确认
- 家庭共享查看与共同编辑

### P2：增强长期陪伴与留存

- 每日回想趋势
- 固定节奏提醒
- 更丰富的生活触发器库
- 家庭群分享摘要

### P3：远期扩展，不作为当前阶段重点

- 社区养老活动场景
- 护理员陪访模式
- 家庭档案批量管理
- 机构级导出模板

由于当前已明确第一阶段做普通家用产品，因此 P3 不应进入近期路线图，也不建议影响当前信息架构。

## 14. 推荐的对外产品表达

可直接用于介绍页的版本：

### 版本 A

岁语帮助老人留下人生故事，也帮助家人更了解父母走过的岁月。  
通过温和访谈、照片回想、人物地图与每日轻量回想，岁语把家庭记忆整理成可以传承的回忆录，也让陪伴变得更具体。

### 版本 B

岁语是一款面向老年人与家属协同使用的回忆录与脑健康陪伴产品。  
它既能整理人生故事，也能通过熟悉的人、地点、照片和时代线索，帮助老人每天轻轻回想。

### 版本 C

岁语主要面向想陪伴父母的子女，也真正服务于老人本人。  
它让家人可以一起整理人生故事、补全人物和照片，也让老人通过轻量回想持续获得熟悉感、陪伴感和表达感。

## 15. 下一步建议

如果按执行优先级推进，建议顺序如下：

1. 先改首页定位与入口，明确“老人本人 / 子女代访 / 日常回想”
2. 再改资料页，补齐家庭协同与关系信息
3. 重做日常回想逻辑，彻底去掉答案比对感
4. 增加人物关系手动校正与家庭共编
5. 最后扩触发器库、趋势记录和家庭分享
